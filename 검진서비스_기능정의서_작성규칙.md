# BizCare 기능정의서 컨벤션

## 1. 컬럼 구조 (13열 고정)

| 열 | 컬럼명 | 설명 |
|---|---|---|
| A | 번호 | 순번 (1, 2, 3…) |
| B | ID | 화면 고유 ID (예: eap-login-01) |
| C | 분류 | 최상위 서비스 분류 (로그인 / 메인 / 검진예약 / 검진결과 / 병원관리자 / 건강자료실) |
| D | 1Depth | 내비게이션 1단계 |
| E | 2Depth | 내비게이션 2단계 |
| F | 3Depth | 내비게이션 3단계 |
| G | 4Depth | 내비게이션 4단계 |
| H | 5Depth | 내비게이션 5단계 |
| I | 화면명 | 실제 화면/컴포넌트 이름 |
| J | 화면 종류 | Page / 블럭 / 필터 / 버튼 / page (소문자 혼용 주의) |
| K | 표시내용(요소/element) | UI 요소 목록 — 번호 기반 |
| L | 내용 | 동작/정책/분기 설명 — 표시내용 번호와 동기화 |
| M | 비고 | 이슈, 검토 항목, 팀 의견 |

---

## 2. ID 명명 규칙

```
{서비스}-{분류}-{순번 2자리}
예: eap-login-01, eap-main-02
```

- 서비스 prefix: `eap` (현재 단일 서비스)
- 분류: login / main / reservation / result / hospital / health
- 순번: 01부터 시작

---

## 3. 표시내용 작성 규칙

### UI 컴포넌트는 반드시 [대괄호] 사용
```
[입력폼] 이름
[CTA 버튼] 로그인
[헬퍼 텍스트] 초기 비밀번호 안내 문구
[링크] 비밀번호 찾기
[리스트] 계정 목록
[카드] 대상자 카드
[드롭다운] 관계 선택
[배너] 예약기간 배너
[블럭] 대상자 목록
[버튼] 대상자 추가
[프로세스 스텝] Step 1~5 인디케이터
[안내 문구] 본인인증 안내
```

### 번호 체계
- 최상위: `1.`, `2.`, `3.`
- 하위: `1-1.`, `1-2.`, `1-3.`
- 표시내용 번호와 내용 번호 반드시 동기화

---

## 4. 내용 작성 규칙

- 표시내용의 번호와 1:1 대응
- 동작(클릭/탭/입력) → 결과 형식으로 기술
- 분기는 숫자 + 조건으로 명시
  ```
  4. 로그인 성공 후 분기
  4-1. 비밀번호 강제 변경 조건 해당 시 → 비밀번호 강제 변경 화면으로 이동
  4-2. 해당 없을 시 → 본인 확인(간편인증) 화면으로 이동
  ```
- 0번은 화면 진입 조건 또는 전제 설명에 사용

## 5. 비고 색상 규칙 (openpyxl 생성 시)

| 색상 | HEX | 사용 조건 |
|---|---|---|
| 주황 | `FFC000` | SAKAK 연동 관련 항목 |
| 노랑 | `FFFF99` | 검토 필요 / 미확정 항목 |
| 하늘 | `BDD7EE` | 교차 참조 / 타 화면 연계 |
| 흰색 (기본) | `FFFFFF` | 변경 없음 |

### 비고 텍스트 패턴 → 색상 자동 판단
- `확인 필요`, `검토 필요`, `TBD`, `재검토` → 노랑 `FFFF99`
- `SAKAK`, `에임스`, `외부 연동` → 주황 `FFC000`
- `별도 확인`, `하위 참조`, `통합 여부` → 하늘 `BDD7EE`

## 6. 화면 종류 분류 기준

| 종류 | 설명 | 예시 |
|---|---|---|
| Page | 독립 전체 화면 | 로그인, 예약 완료 |
| 블럭 | 페이지 내 섹션/컴포넌트 | 대상자 목록, 예약기간 배너 |
| 필터 | 필터/정렬 UI | 병원 목록 필터 |
| 버튼 | 단일 버튼/진입점 | 비교 병원 선택, 단계별 FAQ 버튼 |
