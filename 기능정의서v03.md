# 1. 기능정의서.md (MD파일 분리예정)
## 기능정의서 설명 **(추후 파일 분리 예정)**

#### 스프레드시트 또는 엑셀파일 화면설계서를 분석에서 페이지와 UI를 생성하는 규칙이 포함된 마크다운 문서

### 기능정의서 기본 사용 규칙

1. 스프레드 시트의 행과 열을 잘 사용한다.
2. 행(Row)은 각각 항목을 기록한다.
3. 열(Colulm)은 반복되는 요소를 정리한다.
4. C열 분류는 현재 화면 또는 기능의 그룹등을 정의한다.
5. D열~G열은 메뉴의 상위하위 체계를 정의한다. 메뉴는 트리구조로 되어있다. 
6. H열 5Depth은 페이지 하의의 블록, 기능, 모듈 명 등을 정의한다.


### 컬럼명

1. A열 : 번호,
2. B열 : ScreeenID
3. C열 : 분류
4. D열 : 1 Depth 메뉴/화면명
5. E열 : 2 Depth 메뉴/화면명
6. F열 : 3 Depth 메뉴/화면명
7. G열 : 4 Depth 메뉴/화면명
8. H열 : 5 Depth 영역, 블록
9. I열 : 화면명(Screen Name)
10. J열 : Component Type(컴포넌트 타입), Screen, Popup, Layer Popup, Bottom Sheet, Section, Component 등
11. K열 : Component ID(컴포넌트 ID)
12. L열 : Conponent no (컴포넌트 순번)
13. M열 : 최종ID, CSS 클래스에 추가, 컴포넌트 단위 업데이트시 사용
14. **N열 : 표시내용 컴포넌트(Component)**   
15. **L열  : Property(컴포넌트 속성) N열에 대한 속성 및  추가 기능 정의**
16. P열 : 비고, 해당 컴포넌트의 비고, 사전처리 조건 등
17. Q열 변경일시 : 현재 행에서 수정, 업데이트한 날짜와 시간
18. R열 변경컬럼 : 현재 행에서 변경된 컬럼 목록
19. S열 담당자 : 수정한 담당자 이메일

###  기능정의서 메뉴트리 구조
1. 기능정의서 기능정의 탭에서 1Depth~4Depth 까지 피벗 형태의 트리구조로 되어있다.
2. 화면 종류는 Page, 블록, 팝업, 레이어팝업, 빈페이지 등이 있다.
3. ** 화면 종류에서 블록, 버튼, 텍스트, 콘텐츠는 메뉴 구조에서 제외한다.**
4. 화면 Depth, 화면명, 화면종류를 구분하여 메뉴트리를 순서없는 목록(UL)형태로 마크업한다.

### 기능정의서의 구조
기능정의서에서는 Screen, Section, Component, Element 총 4단계를 사용하되 **Component, Element를 합쳐 3단계로 사용을 권장한다.** Component가 여러개의 Element로 구성된 경우에는 4단계까지 허용한다.
1. Screen(화면) : Page
2. Section(영역) : Layout
3. Component(컴포넌트) : 1개 또는 여러개의 Element로 되어 있다.
    > 1. 목적 : 이 컴포넌트가 해결하는 사용자 과제
    > 2. 입력값 : 사용자가 입력하거나 외부에서 받는 데이터, 기본값, 필수여부
    > 3. 노출조건 : 언제 보이고 언제 숨는지
    > 4. 상태 : default, loading, empty, error, disabled, selected 등
    > 5. 이벤트 : tap, click, change, submit, swipe 등
    > 6. 결과 : 화면 이동, API 호출, 토스트 표시, 값 저장 등
4. Element(요소) : text, image, form, Placeholder, Label, Icon, Validation Message 등
    > 1. Property(속성)
    > 2. State(상태)
    > 3. Event(이벤트)

## 2.1 텍스트 입력 필드 (Component)
 
- **목적**: 사용자가 단문 텍스트를 입력하는 기본 입력 필드. (다른 입력 필드의 상위 정책이 됨)
- **입력값**:
  - 인풋 텍스트 기본 글자수 20글자, 최대(Max) 100글자 (모바일 / PC·웹 공통)
  - 필수 여부: 원문 미기재 (서비스 정책에 따름)
- **노출조건**: 원문 미기재
- **상태**: 활성화(Enabled, 기본값) / 마우스오버(Hovered) / 포커스(Focused) / 오류(Error) / 비활성화(Disabled)
- **이벤트**: 입력(change) / 포커스(focus) / 마우스오버(hover)
- **결과**:
  - 입력 오류 발생 시 서포팅 텍스트에 에러 표시
  - 입력 오류 발생 시 폼을 저장하지 않고, 해당 에러 항목에 포커싱
### Element 구성
1. **레이블 텍스트 (Label Text)**
   - Property: 항목 제목
   - State: 원문 미기재
   - Event: 원문 미기재
2. **인풋 텍스트 (Input Text)**
   - Property: 기본 20글자 / Max 100글자 (모바일·PC·웹 공통)
   - State: 활성화 / 마우스오버 / 포커스 / 오류 / 비활성화
   - Event: change(입력) / focus(포커싱)
3. **테일링 아이콘 (Tailing Icon)**
   - Property: 인풋 우측 아이콘
   - State: 원문 미기재
   - Event: 원문 미기재
4. **서포팅 텍스트 (Supporting Text)**
   - Property: 보조 안내 / 오류 메시지 표시 영역
   - State: 오류(Error) 상태와 연계하여 에러 표시
   - Event: 원문 미기재
5. **플레이스 홀더 (Place Holder)**
   - Property: 지정 시 지정값 사용 / 미지정 시 "{레이블} 입력" 으로 자동 생성
   - State: 원문 미기재
   - Event: 원문 미기재

## 2.2 패스워드 입력 필드 (Component)
 
- **목적**: 사용자의 비밀번호를 입력받는 필드. (텍스트 입력 필드 정책을 상속)
- **입력값**:
  - 최소 8자리 이상 ~ 20자리 이하
  - 영문 + 숫자 + 특수문자 조합 중 3가지 이상 포함 여부 체크
  - 최초 생성 시 2회 입력(1회/2회 입력값 동일 여부 체크)
  - 필수 여부: 원문 미기재 (서비스 정책에 따름)
- **노출조건**: 원문 미기재
- **상태**: 텍스트 입력 필드(2.1)의 상태 정책을 따른다. (활성화 / 마우스오버 / 포커스 / 오류 / 비활성화)
- **이벤트**: 입력(change) / 포커스(focus)
- **결과**:
  - 규칙 위반(자릿수·조합·재입력 불일치) 시 오류 처리 (텍스트 입력 필드 결과 정책 준용)
  - DB 저장 시 암호화 알고리즘 사용 (암호화 방식은 DB 설계 에이전트가 판단)
### Element 구성
1. **레이블 텍스트 (Label Text)** — 텍스트 입력 필드(2.1) 정책 준용
2. **패스워드 인풋 텍스트 (Password Input Text)**
   - Property: 8~20자리 / 영문+숫자+특수문자 3종 이상 조합
   - State: 활성화 / 마우스오버 / 포커스 / 오류 / 비활성화
   - Event: change(입력)
3. **패스워드 재입력 인풋 (Confirm Input Text)** — 최초 생성 시 2회 입력, 1·2회 값 일치 검증
4. **서포팅 텍스트 (Supporting Text)** — 오류 시 에러 표시 연계

## 2.3 날짜 입력 필드 (Component)
 
- **목적**: 사용자가 날짜를 입력/선택하는 필드.
- **입력값**:
  - 형식: 기본 YYYY-MM-DD (예: 2025-01-01), 정책에 따라 YYYY.MM.DD / YYYY/MM/DD 허용
  - 입력 방식: 직접 입력 / 캘린더 픽커 선택
  - 날짜 범위(시작일~종료일) 필요 시 두 개의 날짜 필드를 쌍으로 구성
  - 미래/과거 날짜 제한: 서비스 정책에 따라 설정
- **노출조건**: 원문 미기재
- **상태**: 활성화(기본값) / 마우스오버 / 포커스 / 오류 / 비활성화
- **이벤트**: 직접 입력(change) / 캘린더 아이콘 클릭(click → 달력 팝업) / 포커스(focus)
- **결과**:
  - 유효성 위반(존재하지 않는 날짜, 형식 오류, 종료일이 시작일보다 앞섬) 시 오류 처리
  - 오류 시 서포팅 텍스트에 에러 표시, 폼 저장하지 않고 해당 에러에 포커싱
### Element 구성
1. **레이블 텍스트 (Label Text)**
2. **날짜 인풋 텍스트 (Date Input Text)** — Property: 연/월/일 구분자 포함; State: 활성/호버/포커스/오류/비활성; Event: change
3. **캘린더 아이콘 (Calendar Icon, Tailing Icon)** — Event: click(달력 UI 팝업)
4. **서포팅 텍스트 (Supporting Text)** — 오류 연계
5. **플레이스 홀더 (Place Holder)** — Property: 사용 중인 날짜 형식과 동일 표시 (예: YYYY-MM-DD)

## 2.4 숫자 입력 필드 (Component)
 
- **목적**: 사용자가 숫자 값을 입력하는 필드.
- **입력값**:
  - 최솟값(Min) / 최댓값(Max): 서비스 정책에 따라 설정
  - 기본값(Default): 서비스 정책에 따라 설정, 미설정 시 공란
  - 소수점 허용 여부 및 자릿수: 서비스 정책에 따라 설정
  - 천 단위 구분자(콤마): 금액 등 큰 숫자에 정책에 따라 적용
- **노출조건**: 원문 미기재
- **상태**: 활성화(기본값) / 마우스오버 / 포커스 / 오류 / 비활성화
- **이벤트**: 직접 입력(change, 숫자 외 문자 차단) / 스테퍼 +- 클릭(click, step 단위 증감) / 포커스(focus)
- **결과**:
  - 유효성 위반(숫자 외 문자, Min 미만·Max 초과, 미허용 소수점) 시 오류 처리
  - 오류 시 서포팅 텍스트에 에러 표시, 폼 저장하지 않고 해당 에러에 포커싱
### Element 구성
1. **레이블 텍스트 (Label Text)**
2. **숫자 인풋 텍스트 (Number Input Text)** — Property: 숫자만 입력 / Min·Max·소수점은 정책; State: 활성/호버/포커스/오류/비활성; Event: change
3. **증감 버튼 (Stepper Button, 선택)** — Property: +/- 버튼; Event: click(step 단위 증감)
4. **단위 텍스트 (Unit Text, 선택)** — Property: 원·kg·%·개 등
5. **서포팅 텍스트 (Supporting Text)** — 오류 연계
6. **플레이스 홀더 (Place Holder)** — 예: 숫자 입력

## 2.5 비정렬 목록 (ul, Unordered List) (Component)
 
- **목적**: 순서 의미가 없는 항목들을 불릿 형태로 나열/관리하는 목록.
- **입력값**:
  - 아이템 최소 개수 1개 / 최대 개수는 정책(미설정 시 제한 없음)
  - 아이템당 텍스트: 기본 20글자, Max 100글자 (텍스트 입력 필드 정책 준용)
  - 중첩(Nested) 최대 3단계 허용, 이상은 정책에 따라 설정
- **노출조건**: 원문 미기재
- **상태**: 기본(Default, 불릿+텍스트 표시) / 비활성화(Disabled, 전체 또는 개별 아이템) / 아이템 추가·삭제 가능 여부(정책)
- **이벤트** (동적 목록인 경우): 아이템 추가(click) / 아이템 삭제(click, X) / 순서 변경(드래그앤드롭 또는 이동 버튼, 정책)
- **결과**:
  - 필수 목록은 아이템 1개 이상 있어야 저장 가능
  - 최소/최대 아이템 수 위반 시 오류 처리, 초과 시 추가/삭제 버튼 비활성화
### Element 구성
1. **레이블 텍스트 (Label Text, 선택)** — 목록 전체 제목
2. **목록 아이템 (List Item)** — Property: 불릿 기호 + 텍스트 / 아이템당 20자(Max 100자); State: 기본 / 비활성화
3. **불릿 마커 (Bullet Marker)** — Property: •, -, ▪ 등 디자인 시스템 정의
4. **아이템 추가 버튼** (동적) — Property: 목록 하단 위치(예: "+ 항목 추가"); Event: click
5. **아이템 삭제 버튼(X)** (동적) — Property: 각 아이템 우측; Event: click
6. **서포팅 텍스트 (Supporting Text, 선택)** — 오류 연계

## 2.6 정렬 목록 (ol, Ordered List) (Component)
 
- **목적**: 순서가 의미 있는 항목들을 번호와 함께 나열/관리하는 목록.
- **입력값**:
  - 시작 번호: 기본 1, 정책에 따라 변경 가능
  - 번호 형식: 아라비아(1,2,3) / 로마(i,ii,iii) / 알파벳(a,b,c) 등 정책
  - 아이템 최소 1개 / 최대는 정책(미설정 시 제한 없음)
  - 아이템당 텍스트: 기본 20글자, Max 100글자 (텍스트 입력 필드 정책 준용)
  - 중첩(Nested) 최대 3단계 허용
- **노출조건**: 원문 미기재
- **상태**: 기본(Default, 순서 마커+텍스트) / 비활성화(전체 또는 개별) / 아이템 추가·삭제 가능 여부(정책)
- **이벤트** (동적 목록인 경우): 추가(click) / 삭제(click, X → 번호 자동 재정렬) / 순서 변경(드래그앤드롭·이동 버튼 → 번호 자동 재정렬)
- **결과**:
  - 필수 목록은 아이템 1개 이상 있어야 저장 가능
  - 최소/최대 아이템 수 위반 시 오류 처리, 초과 시 추가/삭제 버튼 비활성화
### Element 구성
1. **레이블 텍스트 (Label Text, 선택)** — 목록 전체 제목
2. **목록 아이템 (List Item)** — Property: 순서 번호 + 텍스트 / 아이템당 20자(Max 100자); State: 기본 / 비활성화
3. **순서 마커 (Order Marker)** — Property: 1,2,3 또는 가,나,다 등 디자인 시스템 정의
4. **아이템 추가 버튼** (동적) — Property: 목록 하단(예: "+ 항목 추가"); Event: click
5. **아이템 삭제 버튼(X)** (동적) — Event: click(삭제 후 번호 자동 재정렬)
6. **서포팅 텍스트 (Supporting Text, 선택)** — 오류 연계

## 2.7 체크박스 (Checkbox) (Component)
 
- **목적**: 단일/다중 항목을 선택·해제하는 입력 요소.
- **입력값**:
  - 단일 선택 또는 다중 선택 모두 가능
  - 그룹 최소 선택 수 / 최대 선택 수: 서비스 정책에 따라 설정
  - 전체 선택(Select All): 그룹인 경우 최상단 제공 가능(정책)
- **노출조건**: 원문 미기재
- **상태**: 활성화(Enabled, 미선택 기본값) / 선택됨(Checked) / 부분 선택(Indeterminate) / 마우스오버(Hovered) / 오류(Error) / 비활성화(Disabled)
- **이벤트**: 마커 또는 레이블 클릭(click → 선택/해제 토글) / 마우스오버(hover)
- **결과**:
  - 필수 체크박스(약관 동의 등) 미선택 시 오류 처리
  - 최소 선택 수 미달 시 오류 처리 / 최대 선택 수 초과 시 선택 차단
  - 오류 시 서포팅 텍스트에 에러 표시, 폼 저장하지 않고 해당 에러에 포커싱
### Element 구성
1. **체크박스 마커 (Checkbox Marker)** — Property: □ / ☑ 아이콘; State: 미선택 / 선택됨 / 부분선택(-) / 호버 / 비활성화; Event: click(토글)
2. **레이블 텍스트 (Label Text)** — Property: 마커 좌·우측 위치; Event: click(토글)
3. **서포팅 텍스트 (Supporting Text, 선택)** — Property: 레이블 하단; State: 오류 연계

## 2.8 라디오버튼 (Radio Button) (Component)
 
- **목적**: 동일 그룹 내에서 상호 배타적으로 하나만 선택하는 입력 요소.
- **입력값**:
  - 그룹 내 항목 수: 최소 2개 이상 (단일 항목은 체크박스 권장)
  - 기본 선택값(Default Selected): 정책에 따라 설정, 미설정 시 미선택
  - 그룹 레이아웃: 세로 나열(기본값) 또는 가로 나열
- **노출조건**: 원문 미기재
- **상태**: 활성화(Enabled, 미선택 기본값) / 선택됨(Selected) / 마우스오버(Hovered) / 오류(Error) / 비활성화(Disabled)
- **이벤트**: 마커 또는 레이블 클릭(click → 선택, 동일 그룹 내 기존 선택 자동 해제) / 마우스오버(hover)
- **결과**:
  - 필수 그룹에서 미선택 시 오류 처리
  - 비활성화 항목만 남은 경우 그룹 전체 비활성화 처리
  - 오류 시 서포팅 텍스트에 에러 표시, 폼 저장하지 않고 해당 에러에 포커싱
  - 단독 선택 해제(클릭 해제)는 기본 불가, 정책에 따라 허용 여부 설정
### Element 구성
1. **라디오 마커 (Radio Marker)** — Property: ○ / ● 아이콘; State: 미선택 / 선택됨 / 호버 / 비활성화; Event: click(선택)
2. **레이블 텍스트 (Label Text)** — Property: 마커 좌·우측 위치; Event: click(선택)
3. **서포팅 텍스트 (Supporting Text, 선택)** — Property: 레이블 하단; State: 오류 연계

## 2.9 전화번호 입력 필드 (Component)
 
- **목적**: 사용자가 전화번호를 입력하는 필드.
- **입력값**:
  - 숫자만 입력 가능(영문·특수문자 차단)
  - 대시(-) 자동 삽입(사용자 직접 입력 불필요)
  - 자동 포맷(한국 기준): 휴대폰 010-XXXX-XXXX(11자리, 3-4-4) / 서울 일반 02-XXXX-XXXX(9~10자리, 2-3or4-4) / 그 외 지역 0XX-XXX-XXXX(10자리, 3-3-4) / 대표번호 XXXX-XXXX(8자리, 4-4)
  - 최대 길이: 대시 포함 13자리 이하
  - 국제번호 지원 시 국가 코드 선택 후 해당 국가 형식 적용
- **노출조건**: 원문 미기재
- **상태**: 활성화(기본값) / 마우스오버 / 포커스 / 오류 / 비활성화
- **이벤트**: 입력(change, 숫자 외 차단 + 대시 자동 포맷) / 국가 코드 선택(선택) / 포커스(focus)
- **결과**:
  - 숫자 외 문자 입력 시 입력 차단(오류 표시 불필요)
  - 자릿수 부족·존재하지 않는 국번·필수 미입력 시 오류 처리
  - 오류 시 서포팅 텍스트에 에러 표시, 폼 저장하지 않고 해당 에러에 포커싱
### Element 구성
1. **레이블 텍스트 (Label Text)**
2. **전화번호 인풋 텍스트 (Phone Input Text)** — Property: 숫자만, 대시 자동 포맷, Max 13자리; State: 활성/호버/포커스/오류/비활성; Event: change
3. **국가 코드 선택 (Country Code Selector, 선택)** — Property: 국제번호 지원 여부는 정책
4. **서포팅 텍스트 (Supporting Text)** — 오류 연계
5. **플레이스 홀더 (Place Holder)** — 예: 010-0000-0000

## 2.10 시간 입력 필드 (Component)
 
- **목적**: 사용자가 시각(시:분)을 입력/선택하는 필드.
- **입력값**:
  - 시간제: 24시간제(기본값) 또는 12시간제(AM/PM), 정책에 따라 설정
  - 분 입력 간격: 1분 / 10분 / 20분 / 30분 중 정책 선택, 미설정 시 1분 간격
  - 시간 범위(Min~Max Time): 선택, 정책에 따라 설정(예: 09:00~18:00)
- **노출조건**: 원문 미기재
- **상태**: 활성화(기본값) / 마우스오버 / 포커스 / 오류 / 비활성화
- **이벤트**: 직접 입력(change) / 시계 아이콘 클릭(click → 타임 픽커 팝업) / 포커스(focus)
- **결과**:
  - 존재하지 않는 시간, 분 간격 불일치, 범위 초과, 형식 오류 시 오류 처리
  - 시간 범위 지정 시 범위 외 선택/입력 차단
  - 오류 시 서포팅 텍스트에 에러 표시, 폼 저장하지 않고 해당 에러에 포커싱
### Element 구성
1. **레이블 텍스트 (Label Text)**
2. **시간 인풋 텍스트 (Time Input Text)** — Property: 시:분 형식 / 분 간격은 정책; State: 활성/호버/포커스/오류/비활성; Event: change
3. **시계 아이콘 (Clock Icon, Tailing Icon)** — Event: click(타임 픽커 팝업)
4. **오전/오후 선택 (AM/PM Selector)** — Property: 12시간제 사용 시
5. **서포팅 텍스트 (Supporting Text)** — 오류 연계
6. **플레이스 홀더 (Place Holder)** — 예: HH:MM

## 2.11 날짜+시간 동시 입력 필드 (DateTime) (Component)
 
- **목적**: 날짜와 시간을 함께 입력/선택하는 필드.
- **입력값**:
  - 입력 방식: 단일 필드(통합 표시) / 분리 필드(날짜·시간 독립) / 픽커(캘린더 → 타임픽커 단계 진행), 정책 선택
  - 날짜 정책: 2.3 날짜 입력 필드 정책 준용 (기본 YYYY-MM-DD)
  - 시간 정책: 2.10 시간 입력 필드 정책 준용 (분 간격 1/10/20/30분, 24/12시간제)
  - 날짜+시간 범위(Min~Max DateTime): 선택, 정책에 따라 설정
- **노출조건**: 원문 미기재
- **상태**: 활성화(기본값) / 마우스오버 / 포커스 / 오류(날짜 또는 시간 중 하나라도 오류) / 비활성화
- **이벤트**: 입력(change) / 아이콘 클릭(click → 날짜 선택 후 시간 선택 단계 자동 이동) / 포커스(focus)
- **결과**:
  - 날짜 유효성(2.3), 시간 유효성(2.10) 준용
  - 필수 항목에서 날짜만/시간만 입력 시 오류 처리
  - 시작 일시 > 종료 일시(동일 날짜는 시간까지 비교) 시 오류 처리
  - 오류 시 서포팅 텍스트에 에러 표시, 폼 저장하지 않고 해당 에러에 포커싱
### Element 구성
1. **레이블 텍스트 (Label Text)**
2. **날짜 인풋 텍스트 (Date Input Text)** — Property: YYYY-MM-DD (2.3 준용)
3. **시간 인풋 텍스트 (Time Input Text)** — Property: HH:MM (2.10 준용)
4. **캘린더+시계 아이콘 (DateTime Icon, Tailing Icon)** — Event: click(날짜→시간 단계 픽커)
5. **서포팅 텍스트 (Supporting Text)** — 오류 연계
6. **플레이스 홀더 (Place Holder)** — 예: YYYY-MM-DD HH:MM
