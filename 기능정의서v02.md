# 2. 기능정의서.md
## 기능정의서 설명 **(추후 파일 분리 예정)**

#### 스프레드시트 또는 엑셀파일 화면설계서를 분석에서 페이지와 UI를 생성하는 규칙이 포함된 마크다운 문서

### 기능정의서 기본 사용 규칙

1. 스프레드 시트의 행과 열을 잘 사용한다.
2. 행(Row)은 각각 항목을 기록한다.
3. 열(Colulm)은 반복되는 요소를 정리한다.
4. C열 분류는 현재 화면 또는 기능의 그룹등을 정의한다.
5. D열~G열은 메뉴의 상위하위 체계를 정의한다. 메뉴는 트리구조로 되어있다. 
6. H열 5Depth은 페이지 하의의 블록, 기능, 모듈 명 등을 정의한다.
7. I열은 화면명을 정의한다
8. J열 화면 종류 : 페이지(화면), 레이어팝업, 풀팝업, 바텀시트, 빈페이지 등이 있다.
9. **K열 표시내용(요소/Element) : 주요 UI 및 요소(Elememt) 항목들에 대한 내용이 정의되어있다.**
10. **L열 내용 : K열 요소에 대한 상세한 내용이 기술 되어 있다.**
11. M열 비고 : 기능적으로 추가되어야 하는 체크사항이나 항목이 기술 되어 있다.

### 컬럼명
1. A열 : 번호,
2. B열 : ID
3. C열 : 분류
4. D열 : 1Depth 메뉴
5. E열 : 2 Depth 메뉴
6. F열 : 3 Depth 메뉴
7. G열 : 4 Depth 메뉴
8. H열 : 5 Depth 모듈명 또는 기능명, 블록 제목 등
9. I열 : 화면명(PageID)
10. J열 : 화면 종류
11. **K열 : 표시내용 요소 (Element)**   
12. **L열 :  내용은 K열에 대한 추가 기능 정의**
13. M열 : 비고

###  기능정의서 메뉴트리 구조
1. 기능정의서 기능정의 탭에서 1Depth~4Depth 까지 피벗 형태의 트리구조로 되어있다.
2. 화면 종류는 Page, 블록, 팝업, 레이어팝업, 빈페이지 등이 있다.
3. ** 화면 종류에서 블록, 버튼, 텍스트, 콘텐츠는 메뉴 구조에서 제외한다.**
4. 화면 Depth, 화면명, 화면종류를 구분하여 메뉴트리를 순서없는 목록(UL)형태로 마크업한다.

### 기능정의서의 구조
1. Screen(화면) : 
2. Section(영역)
3. Component(컴포넌트)
    > 1. 목적 : 이 컴포넌트가 해결하는 사용자 과제
    > 2. 입력값 : 사용자가 입력하거나 외부에서 받는 데이터
    > 3. 노출조건 : 언제 보이고 언제 숨는지
    > 4. 상태 : default, loading, empty, error, disabled, selected 등
    > 5. 이벤트 : tap, click, change, submit, swipe 등
    > 6. 결과 : 화면 이동, API 호출, 토스트 표시, 값 저장 등
4. Element(요소)
    > 1. Property(속성)
    > 2. State(상태)
    > 3. Event(이벤트)

### 2.3 날짜 입력 필드
 
1. 날짜 입력 필드 구조:
    1. 레이블 텍스트 : Label Text
    2. 날짜 인풋 텍스트 : Date Input Text (연/월/일 구분자 포함)
    3. 캘린더 아이콘 : Calendar Icon (Tailing Icon)
    4. 서포팅 텍스트 : Supporting Text
    5. 플레이스 홀더 : Place Holder (예: YYYY-MM-DD)
2. 날짜 입력 방식:
    1. 직접 입력 : 키보드로 날짜 직접 입력 가능
    2. 캘린더 픽커 : 캘린더 아이콘 클릭 시 달력 UI 팝업으로 날짜 선택 가능
    3. 날짜 범위 선택 : 시작일 ~ 종료일 범위 지정이 필요한 경우 두 개의 날짜 필드를 쌍으로 구성
3. 날짜 형식:
    1. 기본 형식 : YYYY-MM-DD (예: 2025-01-01)
    2. 서비스 정책에 따라 YYYY.MM.DD 또는 YYYY/MM/DD 형식 허용
    3. 플레이스 홀더는 사용 중인 날짜 형식과 동일하게 표시 (예: YYYY-MM-DD)
4. 날짜 필드 상태 : 활성화(기본값), 비활성화, 에러, 포커싱, 호버
    1. 활성화(Enabled) : 기본값
    2. 마우스오버(Hovered) : 마우스 오버
    3. 포커스(Focused) : 포커싱, 또는 커서 위치, 입력 중 가능한 상태
    4. 오류(Error, 에러) : 입력 오류가 있는 경우, 서포팅 텍스트와 연계
    5. 비활성화(Disabled) : 비활성화, 사용안함 처리된 상태
5. 날짜 유효성 검증:
    1. 존재하지 않는 날짜 입력 시 오류 처리 (예: 2025-02-30)
    2. 형식이 맞지 않는 경우 오류 처리
    3. 날짜 범위가 지정된 경우 종료일이 시작일보다 앞서면 오류 처리
    4. 미래 날짜 또는 과거 날짜 제한이 필요한 경우 서비스 정책에 따라 설정
6. 입력 오류 발생시 서포팅 텍스트에서 에러 표시
7. 입력 오류 발생시 폼을 저장하지 않고 에러 표시 및 해당 에러에 포커싱

### 2.4 숫자 입력 필드
 
1. 숫자 입력 필드 구조:
    1. 레이블 텍스트 : Label Text
    2. 숫자 인풋 텍스트 : Number Input Text
    3. 증감 버튼 (선택) : Stepper Button (+ / - 버튼, 필요 시 사용)
    4. 단위 텍스트 (선택) : Unit Text (예: 원, kg, %, 개)
    5. 서포팅 텍스트 : Supporting Text
    6. 플레이스 홀더 : Place Holder (예: 숫자 입력)
2. 숫자 입력 디폴트값:
    1. 최솟값(Min), 최댓값(Max) : 서비스 정책에 따라 설정
    2. 기본값(Default) : 서비스 정책에 따라 설정, 미설정 시 공란
    3. 소수점 : 허용 여부 및 소수점 자릿수는 서비스 정책에 따라 설정
3. 숫자 입력 방식:
    1. 직접 입력 : 키보드로 숫자 직접 입력 가능, 숫자 이외 문자는 입력 차단
    2. 스테퍼 : +/- 버튼 사용 시 설정된 단위(step)만큼 증감
    3. 천 단위 구분자(콤마) : 금액 등 큰 숫자의 경우 서비스 정책에 따라 표시 적용
4. 숫자 필드 상태 : 활성화(기본값), 비활성화, 에러, 포커싱, 호버
    1. 활성화(Enabled) : 기본값
    2. 마우스오버(Hovered) : 마우스 오버
    3. 포커스(Focused) : 포커싱, 또는 커서 위치, 입력 중 가능한 상태
    4. 오류(Error, 에러) : 입력 오류가 있는 경우, 서포팅 텍스트와 연계
    5. 비활성화(Disabled) : 비활성화, 사용안함 처리된 상태
5. 숫자 유효성 검증:
    1. 숫자 이외의 문자 입력 시 오류 처리
    2. 최솟값 미만 또는 최댓값 초과 시 오류 처리
    3. 소수점 미허용 필드에 소수점 입력 시 오류 처리
6. 입력 오류 발생시 서포팅 텍스트에서 에러 표시
7. 입력 오류 발생시 폼을 저장하지 않고 에러 표시 및 해당 에러에 포커싱

### 2.5 비정렬 목록 (ul, Unordered List)
 
1. 비정렬 목록 구조:
    1. 레이블 텍스트 : Label Text (목록 전체에 대한 제목, 선택)
    2. 목록 아이템 : List Item (불릿 기호 + 텍스트)
    3. 불릿 마커 : Bullet Marker (•, -, ▪ 등 디자인 시스템 정의에 따름)
    4. 서포팅 텍스트 : Supporting Text (선택)
2. 비정렬 목록 디폴트값:
    1. 아이템 최소 개수 : 1개
    2. 아이템 최대 개수 : 서비스 정책에 따라 설정, 미설정 시 제한 없음
    3. 아이템당 텍스트 : 기본사이즈 20글자, Max 100글자 (텍스트 입력 필드 정책 준용)
    4. 중첩(Nested) 목록 : 최대 3단계까지 허용, 이상은 서비스 정책에 따라 설정
3. 비정렬 목록 상태:
    1. 기본(Default) : 불릿 마커와 텍스트 표시
    2. 비활성화(Disabled) : 목록 전체 또는 개별 아이템 비활성화 가능
    3. 아이템 추가/삭제 가능 여부 : 서비스 정책에 따라 설정
4. 동적 목록 (아이템 추가/삭제 가능한 경우):
    1. 아이템 추가 버튼 : 목록 하단에 위치 (예: "+ 항목 추가")
    2. 아이템 삭제 버튼 : 각 아이템 우측에 삭제(X) 버튼 제공
    3. 아이템 순서 변경 : 드래그앤드롭 또는 이동 버튼으로 순서 변경 가능 (서비스 정책에 따라 설정)
    4. 최소/최대 아이템 수 초과 시 추가/삭제 버튼 비활성화
5. 유효성 검증:
    1. 필수 목록인 경우 아이템이 1개 이상 있어야 저장 가능
    2. 최소/최대 아이템 수 위반 시 오류 처리

### 2.7 체크박스 (Checkbox)

    1. 체크박스 마커 : Checkbox Marker (□ / ☑ 아이콘)
    2. 레이블 텍스트 : Label Text (체크박스 우측 또는 좌측에 위치)
    3. 서포팅 텍스트 : Supporting Text (선택, 레이블 하단에 위치)
2. 체크박스 선택 방식:
    1. 단일 선택 또는 다중 선택 모두 가능
    2. 체크박스 마커 또는 레이블 텍스트 클릭 시 선택/해제 토글
    3. 전체 선택(Select All) : 그룹 체크박스인 경우 최상단에 전체 선택 체크박스 제공 가능 (서비스 정책에 따라 설정)
    4. 부분 선택(Indeterminate) : 전체 선택 체크박스에서 일부만 선택된 경우 중간 상태(-) 표시
3. 체크박스 상태 : 활성화(기본값), 선택됨, 부분선택, 비활성화, 에러, 호버
    1. 활성화(Enabled) : 기본값, 미선택 상태
    2. 선택됨(Checked) : 체크 완료 상태
    3. 부분 선택(Indeterminate) : 그룹 내 일부만 선택된 상태
    4. 마우스오버(Hovered) : 마우스 오버
    5. 오류(Error, 에러) : 필수 항목 미선택 등 오류가 있는 경우, 서포팅 텍스트와 연계
    6. 비활성화(Disabled) : 비활성화, 사용안함 처리된 상태 (선택됨/미선택 모두 가능)
4. 체크박스 그룹 설정:
    1. 최소 선택 수 : 서비스 정책에 따라 설정 (예: 1개 이상 필수 선택)
    2. 최대 선택 수 : 서비스 정책에 따라 설정 (예: 최대 3개 선택 가능)
    3. 최대 선택 수 초과 시 추가 선택 불가 처리
5. 유효성 검증:
    1. 필수 체크박스(약관 동의 등) 미선택 시 오류 처리
    2. 최소 선택 수 미달 시 오류 처리
    3. 최대 선택 수 초과 시 선택 차단
6. 입력 오류 발생시 서포팅 텍스트에서 에러 표시
7. 입력 오류 발생시 폼을 저장하지 않고 에러 표시 및 해당 에러에 포커싱

### 2.8 라디오버튼 (Radio Button)
 
1. 라디오버튼 구조:
    1. 라디오 마커 : Radio Marker (○ / ● 아이콘)
    2. 레이블 텍스트 : Label Text (라디오버튼 우측 또는 좌측에 위치)
    3. 서포팅 텍스트 : Supporting Text (선택, 레이블 하단에 위치)
2. 라디오버튼 선택 방식:
    1. 동일 그룹 내에서 단 하나만 선택 가능 (상호 배타적 선택)
    2. 라디오 마커 또는 레이블 텍스트 클릭 시 선택
    3. 선택 해제 : 동일 그룹 내 다른 항목 선택 시 기존 선택 자동 해제
    4. 단독 선택 해제(클릭으로 해제) : 기본적으로 불가, 서비스 정책에 따라 허용 여부 설정
3. 라디오버튼 상태 : 활성화(기본값), 선택됨, 비활성화, 에러, 호버
    1. 활성화(Enabled) : 기본값, 미선택 상태
    2. 선택됨(Selected) : 선택 완료 상태
    3. 마우스오버(Hovered) : 마우스 오버
    4. 오류(Error, 에러) : 필수 항목 미선택 등 오류가 있는 경우, 서포팅 텍스트와 연계
    5. 비활성화(Disabled) : 비활성화, 사용안함 처리된 상태 (선택됨/미선택 모두 가능)
4. 라디오버튼 그룹 설정:
    1. 그룹 내 항목 수 : 최소 2개 이상 (단일 항목은 체크박스 사용 권장)
    2. 기본 선택값(Default Selected) : 서비스 정책에 따라 설정, 미설정 시 미선택 상태
    3. 그룹 레이아웃 : 세로 나열(Vertical, 기본값) 또는 가로 나열(Horizontal) 선택 가능
5. 유효성 검증:
    1. 필수 라디오 그룹에서 아무것도 선택하지 않은 경우 오류 처리
    2. 비활성화 항목만 남은 경우 그룹 전체 비활성화 처리
6. 입력 오류 발생시 서포팅 텍스트에서 에러 표시
7. 입력 오류 발생시 폼을 저장하지 않고 에러 표시 및 해당 에러에 포커싱

### 2.9 전화번호 입력 필드
 
1. 전화번호 입력 필드 구조:
    1. 레이블 텍스트 : Label Text
    2. 전화번호 인풋 텍스트 : Phone Input Text
    3. 국가 코드 선택 (선택) : Country Code Selector (서비스 정책에 따라 국제번호 지원 여부 설정)
    4. 서포팅 텍스트 : Supporting Text
    5. 플레이스 홀더 : Place Holder (예: 010-0000-0000)
2. 전화번호 입력 방식:
    1. 숫자만 입력 가능 : 숫자 이외의 문자(영문, 특수문자 등) 입력 차단
    2. 대시(-) 자동 삽입 : 숫자 입력 시 번호 구간에 맞춰 대시(-) 자동 추가, 사용자가 직접 대시 입력 불필요
    3. 자동 포맷 규칙 (한국 기준):
        - 휴대폰 : 010-XXXX-XXXX (총 11자리 숫자, 형식: 3-4-4)
        - 서울 일반전화 : 02-XXXX-XXXX (총 9~10자리 숫자, 형식: 2-3or4-4)
        - 그 외 지역 일반전화 : 0XX-XXX-XXXX (총 10자리 숫자, 형식: 3-3-4)
        - 대표번호(1588, 1544 등) : XXXX-XXXX (총 8자리 숫자, 형식: 4-4)
    4. 입력 최대 길이 : 대시(-) 포함 13자리 이하 (휴대폰 기준 010-0000-0000)
    5. 국제번호 지원 시 : 국가 코드 선택 후 해당 국가 번호 형식에 맞게 포맷 적용
3. 전화번호 필드 상태 : 활성화(기본값), 비활성화, 에러, 포커싱, 호버
    1. 활성화(Enabled) : 기본값
    2. 마우스오버(Hovered) : 마우스 오버
    3. 포커스(Focused) : 포커싱, 또는 커서 위치, 입력 중 가능한 상태
    4. 오류(Error, 에러) : 입력 오류가 있는 경우, 서포팅 텍스트와 연계
    5. 비활성화(Disabled) : 비활성화, 사용안함 처리된 상태
4. 전화번호 유효성 검증:
    1. 숫자 이외 문자 입력 시 입력 차단 (오류 표시 불필요)
    2. 전화번호 자릿수가 부족한 경우 오류 처리 (예: 010-123 상태로 저장 시도)
    3. 존재하지 않는 국번 형식인 경우 오류 처리 (서비스 정책에 따라 설정)
    4. 필수 항목인 경우 미입력 시 오류 처리
5. 입력 오류 발생시 서포팅 텍스트에서 에러 표시
6. 입력 오류 발생시 폼을 저장하지 않고 에러 표시 및 해당 에러에 포커싱

### 2.10 시간 입력 필드
 
1. 시간 입력 필드 구조:
    1. 레이블 텍스트 : Label Text
    2. 시간 인풋 텍스트 : Time Input Text (시:분 형식)
    3. 시계 아이콘 : Clock Icon (Tailing Icon)
    4. 오전/오후 선택 : AM/PM Selector (12시간제 사용 시)
    5. 서포팅 텍스트 : Supporting Text
    6. 플레이스 홀더 : Place Holder (예: HH:MM)
2. 시간 입력 방식:
    1. 직접 입력 : 키보드로 시간 직접 입력 가능
    2. 타임 픽커 : 시계 아이콘 클릭 시 시간 선택 UI 팝업 제공
    3. 시간제 : 24시간제(기본값) 또는 12시간제(AM/PM) 선택, 서비스 정책에 따라 설정
3. 분 입력 간격 설정:
    1. 분 간격은 서비스 정책에 따라 아래 중 선택하여 적용
        - 1분 간격 : 00, 01, 02, ... 59 (전체 분 선택 가능)
        - 10분 간격 : 00, 10, 20, 30, 40, 50
        - 20분 간격 : 00, 20, 40
        - 30분 간격 : 00, 30
    2. 타임 픽커 사용 시 설정된 분 간격에 해당하는 항목만 선택 가능하도록 표시
    3. 직접 입력 시 설정된 분 간격에 해당하지 않는 값 입력 시 오류 처리
    4. 분 간격 기본값 : 서비스 정책에 따라 설정, 미설정 시 1분 간격 적용
4. 시간 범위 설정 (선택):
    1. 시작 시간(Min Time), 종료 시간(Max Time) : 서비스 정책에 따라 설정
    2. 시간 범위가 지정된 경우 해당 범위 외 시간 선택/입력 차단
    3. 운영 시간 제한이 필요한 경우 적용 (예: 09:00 ~ 18:00만 선택 가능)
5. 시간 필드 상태 : 활성화(기본값), 비활성화, 에러, 포커싱, 호버
    1. 활성화(Enabled) : 기본값
    2. 마우스오버(Hovered) : 마우스 오버
    3. 포커스(Focused) : 포커싱, 또는 커서 위치, 입력 중 가능한 상태
    4. 오류(Error, 에러) : 입력 오류가 있는 경우, 서포팅 텍스트와 연계
    5. 비활성화(Disabled) : 비활성화, 사용안함 처리된 상태
6. 시간 유효성 검증:
    1. 존재하지 않는 시간 입력 시 오류 처리 (예: 25:00, 12:61)
    2. 설정된 분 간격에 맞지 않는 분 값 입력 시 오류 처리
    3. 시간 범위가 설정된 경우 범위 초과 시 오류 처리
    4. 형식이 맞지 않는 경우 오류 처리
7. 입력 오류 발생시 서포팅 텍스트에서 에러 표시
8. 입력 오류 발생시 폼을 저장하지 않고 에러 표시 및 해당 에러에 포커싱

### 2.11 날짜+시간 동시 입력 필드 (DateTime)
 
1. 날짜+시간 입력 필드 구조:
    1. 레이블 텍스트 : Label Text
    2. 날짜 인풋 텍스트 : Date Input Text (YYYY-MM-DD)
    3. 시간 인풋 텍스트 : Time Input Text (HH:MM)
    4. 캘린더+시계 아이콘 : DateTime Icon (Tailing Icon)
    5. 서포팅 텍스트 : Supporting Text
    6. 플레이스 홀더 : Place Holder (예: YYYY-MM-DD HH:MM)
2. 날짜+시간 입력 방식:
    1. 단일 필드 방식 : 날짜와 시간을 하나의 입력 필드에 통합 표시 (예: 2025-01-01 09:00)
    2. 분리 필드 방식 : 날짜 필드와 시간 필드를 각각 독립적으로 구성, 서비스 정책에 따라 선택
    3. 픽커 방식 : 아이콘 클릭 시 날짜 선택(캘린더) → 시간 선택(타임픽커) 순서로 단계적 진행
    4. 날짜 선택 후 시간 선택 단계로 자동 이동 (픽커 사용 시)
3. 날짜 입력 정책:
    1. 2.3 날짜 입력 필드의 정책을 따른다.
    2. 날짜 형식 : YYYY-MM-DD (기본값), 서비스 정책에 따라 변경 가능
4. 시간 입력 정책:
    1. 2.10 시간 입력 필드의 정책을 따른다.
    2. 분 간격 : 서비스 정책에 따라 1분 / 10분 / 20분 / 30분 간격 중 선택 적용
    3. 시간제 : 24시간제(기본값) 또는 12시간제(AM/PM), 서비스 정책에 따라 설정
5. 날짜+시간 범위 설정 (선택):
    1. 시작 일시(Min DateTime), 종료 일시(Max DateTime) : 서비스 정책에 따라 설정
    2. 시작 일시와 종료 일시를 쌍으로 입력하는 경우 종료 일시가 시작 일시보다 앞서면 오류 처리
    3. 동일 날짜의 경우 시간 비교까지 수행 (예: 2025-01-01 09:00 ~ 2025-01-01 08:00 → 오류)
6. 날짜+시간 필드 상태 : 활성화(기본값), 비활성화, 에러, 포커싱, 호버
    1. 활성화(Enabled) : 기본값
    2. 마우스오버(Hovered) : 마우스 오버
    3. 포커스(Focused) : 포커싱, 또는 커서 위치, 입력 중 가능한 상태
    4. 오류(Error, 에러) : 날짜 또는 시간 중 하나라도 오류가 있는 경우, 서포팅 텍스트와 연계
    5. 비활성화(Disabled) : 비활성화, 사용안함 처리된 상태
7. 날짜+시간 유효성 검증:
    1. 날짜 유효성 : 2.3 날짜 입력 필드의 유효성 검증 정책을 따른다.
    2. 시간 유효성 : 2.10 시간 입력 필드의 유효성 검증 정책을 따른다.
    3. 날짜만 입력하고 시간을 입력하지 않은 경우 오류 처리 (필수 항목인 경우)
    4. 시간만 입력하고 날짜를 입력하지 않은 경우 오류 처리 (필수 항목인 경우)
    5. 날짜+시간 범위 위반 시 오류 처리
8. 입력 오류 발생시 서포팅 텍스트에서 에러 표시
9. 입력 오류 발생시 폼을 저장하지 않고 에러 표시 및 해당 에러에 포커싱   

### 2.3 로그인 버튼 **(추후 파일 분리 예정)**

2. 로그인 사전입력 항목 : 아이디, 패스워드, 이름, 보안코드 등 상황에 따라 로그인 입력항목은 변경될 수 있음
3. 로그인 조건이 충족되면 로그인 버튼 활성화됨
