# AIMMED 운영 포털 (어드민) 디자인 시스템
 
> v1 | 2026-06-22
> 이수앱지스 관리자 "운영 포털" 어드민 웹 화면 샘플(메인 대시보드 / 회원 목록 / 회원 목록 상세-기본정보 / 회원 목록 상세-투약이력)을 시각 분석하여 정리한 디자인 가이드.

## 3. 글로벌 구조
 
```
.app
 ├── .utility-header     (흰색 상단 바: 로고 / 관리자명 / 사용자 메뉴)
 ├── .primary-nav        (파란 네비게이션 밴드: 운영 포털)
 ├── .breadcrumb         (운영 포털 > 회원 > 회원 목록 …)
 ├── .page-title         (대제목 + 블루 액센트)
 ├── .content-card       (흰색 둥근 컨테이너 — 실제 화면 영역)
 └── .footer             (주소 / TEL·FAX / 카피라이트)
```
 
### 3-1. 유틸리티 헤더 (.utility-header)
- 배경 흰색, 높이 약 64px, 하단 옅은 구분선
- **좌측**: `AIMMED` 워드마크(블루) + "healthcare for life" 영문 태그라인 + `이수앱지스 관리자` + `고객사` 다크 네이비 pill 뱃지
- **우측**: 알림 벨 아이콘(빨간 카운트 뱃지) · `김수진 (편집가능)` · 🔒`비밀번호 변경` · `로그아웃` (라이트 pill 버튼)
### 3-2. Primary 네비게이션 바 (.primary-nav)
- 풀폭 블루 밴드(밝은 블루 `#2563EB` 계열)
- 현재 메뉴 `운영 포털`은 더 진한 네이비 탭 + 아이콘으로 강조
### 3-3. 브레드크럼 (.breadcrumb)
- 우측 정렬, 회색 소형 텍스트, 🏠 home 아이콘 + `>` 구분
- 예: `운영 포털 > 회원 > 회원 목록 > 회원 목록 상세`
### 3-4. 페이지 타이틀 (.page-title)
- 굵은 대제목(약 24px/700) + 상단 짧은 **블루 액센트 바**(작은 사각형)
### 3-5. 푸터 (.footer)
- 연한 회색 소형 텍스트
- 주소 / `TEL : 02-3015-7500 / FAX : 02-545-0209` / `Copyright ⓒ2025 AIMMED. All right reserved.`

## 6. 컴포넌트 규격
 
### 6-1. KPI / 통계 카드
| 항목 | 값 |
|------|-----|
| Radius | 약 16px |
| 패딩 | 약 20–24px |
| 구성 | (라벨) + **큰 수치 + 단위** + 보조 캡션 + 우측 일러스트 아이콘 |
| 솔리드형 | 핵심 지표 1개를 `--p`(또는 핑크) 솔리드 + 흰색 텍스트로 강조 |
| 라이트형 | 나머지는 파스텔 배경(블루/그린/틸/핑크) + 다크 텍스트 |
- 대시보드: 회원 현황 4종 + 주요 현황 3종(미답변 문의는 핑크 강조) + 탈퇴 현황 3종
- 상세: 투약진료 현황 요약 3종 / 문의 현황 요약 3종
### 6-2. 섹션 카드
| 항목 | 값 |
|------|-----|
| Radius / 배경 | 약 16px / 흰색 |
| 패딩 | 약 24px |
| 섹션 타이틀 | `■`(블루 사각) + 제목(16–18px/700) |
| 우상단 액션 | `전체 보기 >` 텍스트 링크 |
 
### 6-3. 테이블 (.data-table)
| 항목 | 값 |
|------|-----|
| 헤더 배경 | 로즈/핑크 `--table-head` |
| 헤더 텍스트 | 13px/500, 가운데 정렬 |
| 셀 정렬 | 가운데 정렬, 13px |
| 행 높이 | 약 36–40px |
| 행 구분선 | 1px `--border`(하단) |
| 질환명 셀 | 색상 텍스트(파브리=그린 / 고셔=틸) |
| 비고 컬럼 | `상세 보기` 소형 아웃라인 버튼 |
| 우상단 | `엑셀 다운로드` 아웃라인 버튼 + `전체 : N` 카운트 |
 
### 6-4. 라벨-값 정보 테이블 (상세-기본정보)
- 2열 그리드(좌: 기본/가입 정보, 우: 질환 진료/보호자 정보)
- **라벨 셀**: 옅은 틴트 배경 + 회색 텍스트(13px) / **값 셀**: 흰색 배경 + 본문 텍스트(14px)
- 제목 옆 채널 뱃지(예: `카카오싱크`) 노출 가능
### 6-5. 필터 / 검색 패널 (회원 목록)
- 박스형 영역, 행 단위 구성:
  - `가입유형` (라디오: 전체/본인/보호자)
  - `질환명` (라디오: 전체/파브리병/고셔병)
  - `가입일` (라디오: 전체/1·3·6개월/1년/기간선택 + **날짜 범위 피커 2개**)
  - `검색어` (셀렉트: 환자명 + 텍스트 인풋)
- 우측: `검색`(Primary 솔리드) + `초기화`(아웃라인) 버튼
### 6-6. 버튼
| 종류 | 스타일 | 예시 |
|------|--------|------|
| Primary 솔리드 | bg `--p` / 흰색 텍스트 / radius 약 8px | `검색` |
| 아웃라인(블루) | 흰색 bg / 블루 보더·텍스트 | `상세 보기`, `초기화`, `엑셀 다운로드`, `목록`, `전체 보기` |
| 위험(아웃라인) | 흰색 bg / `--danger` 보더·텍스트 | `탈퇴 처리` |
| 유틸리티 pill | 라이트 회색 라운드 | `로그아웃` |
- min-height 약 36–44px, 텍스트 가운데 정렬
### 6-7. 탭 (상세 화면)
- 항목: `기본 정보 / 투약 이력 / 진료 일정 / 문의 내역`
- 활성: 블루 텍스트 + **하단 2px 블루 보더** / 비활성: 회색 텍스트
- 탭 전환 시 하단 콘텐츠 영역 교체
### 6-8. 뱃지 / 칩
| 종류 | 스타일 |
|------|--------|
| 고객사 | 다크 네이비 pill |
| 카카오싱크 | `#FEE500` 옐로우 pill |
| 신규 | 라이트 블루 pill / 블루 텍스트 |
| 파브리병·고셔병 | 질환 색상 칩(그린/틸) |
- radius 약 6–20px(라운드), padding 약 2–4px 8–10px
### 6-9. 차트 (투약 소요시간 추이)
- 세로 막대 그래프, 최근 8회 기준
- 막대: 라이트 블루(`--pl` 계열), **현재/최신 막대만 솔리드 블루** 강조
- 각 막대 상단 값 라벨(예: 90분), x축 날짜 라벨(12-01 … 03-17)

## 7. 화면별 구성 요약
 
### 7-1. 메인 대시보드 (`01_메인대시보드`)
1. 인사 영역: `안녕하세요, 김수진 담당자님` + 기준일(`2026년 4월 16일(화) 기준`)
2. **회원 현황** KPI 4: 전체 활성 회원(솔리드 블루) / 이번 달 신규 / 파브리병 / 고셔병
3. **주요 현황** KPI 3: 미답변 문의(핑크 강조) / 이번 주 진료 예정 / 이번 달 투약
4. **운영 현황 한눈에 보기** 2열: 미답변 문의 리스트 / 이번 주 진료 일정 리스트 (빈 상태: "현재 미답변 문의가 없습니다." 등 placeholder)
5. **최근 신규 가입 회원** 리스트 / **탈퇴 현황** KPI 3(누적·이번 달·탈퇴율, 라이트 핑크 카드)
### 7-2. 회원 목록 (`02_회원목록`)
1. 상단 KPI 4: 전체 가입자 / 이번 달 신규 / 본인 가입 / 보호자 가입
2. 필터·검색 패널(6-5)
3. 회원 목록 테이블: No / 환자명 / 생년월일 / 성별 / 보호자(앱 사용자) / 가입유형 / 질환명 / 가입일 / 비고(상세 보기)
4. 우상단 `엑셀 다운로드`
### 7-3. 회원 목록 상세 - 기본 정보 (`02_…기본_정보`)
1. 환자 헤더: `환자명 : 이서연 (본인 가입)` + `파브리병` 뱃지 + `탈퇴 처리`(위험 버튼)
2. 탭(6-7) — 기본 정보 활성
3. 라벨-값 2열: 기본 정보(카카오싱크) / 질환 진료 정보 / 가입 정보 / 보호자 정보
4. 투약진료 현황 요약 KPI 3 / 문의 현황 요약 KPI 3
5. 하단 `목록` 버튼
### 7-4. 회원 목록 상세 - 투약 이력 (`02_…투약_이력`)
1. 투약진료 현황 요약 KPI 3
2. 투약 이력 테이블: 투약일 / 소요시간 / 몸무게 / 용량 / 병원명 / 진료과 / 주치의 / 투약 간격 / 비고
   - `투약 간격` 셀 강조 색(빨강), `첫투약` 표기
3. 투약 소요시간 추이 막대 차트(6-9)
4. 하단 `목록` 버튼

## 8. 확인 필요 사항 (디자인 소스로 교정 권장)
 
> 아래는 화면 캡처만으로는 정확히 단정할 수 없는 항목입니다. 실제 CSS/피그마가 있으면 그 값으로 교체하세요.
 
1. **정확한 HEX 값** — 본 문서의 컬러는 렌더 추정치. Primary 블루가 `#2563EB`인지 `#1760C4`인지, 질환/핑크 계열의 정확값 확인 필요.
2. **타이포 스케일·폰트 패밀리** — Noto Sans KR / Pretendard 여부 및 px 값 확정.
3. **radius·padding·그림자 토큰** — 카드/버튼/뱃지의 실제 수치.
4. **반응형 기준** — 최대폭, 브레이크포인트(태블릿 대응 여부) 미확인.
